Gemini CLI는 어떻게 파일을 읽고 판단해서 수정하는 것인가?  
그것의 답은 Tools(도구) 이다.

오늘은 Gemini CLI의 핵심이자 AI의 실행력 그 자체인 **도구(Tools)**에 대해 정리했다.

---

## 1. 도구(Tools)란?

Gemini CLI를 사용하다보면 콘솔에 다음과 같은 문구를 확인하게 된다.  
![1.png](../assets/img/posts/2026-04-06-gemini-cli-tools-guide/1.png)
![2.png](../assets/img/posts/2026-04-06-gemini-cli-tools-guide/2.png)  
이것이 Gemini CLI가 Tools를 사용하는 것이다.

Gemini CLI에서 Tools는 **AI가 외부 세계와 상호작용하기 위해 사용하는 도구 역할**을 한다. 

"이 코드를 수정해야겠다"라고 **판단**하면, 실제로 파일 시스템에 접근하여 내용을 바꾸는 **행동**은 바로 이 'Tools'를 통해 이루어진다.  
내가 원하는 Tools를 쓰라고 지시할수도 있고, Gemini CLI가 판단하에 사용하기도 한다.

---

## 2. Gemini CLI의 16가지 기본 Tools

Gemini CLI가 제공하는 주요 Tools들을 역할별로 묶어 정리했다.  
각 Tools는 AI가 문제를 해결하는 과정에서 적재적소에 활용된다.  

터미널 채팅창에 다음 명령어를 입력하면 볼수있는 목록이다.
```bash
> /tools
```
현재 활성화된 모든 도구의 이름과 구체적인 설명이 리스트로 출력된다.

### 🕵️‍♂️ 코드 분석 및 탐색 (Investigation)
코드를 고치기 전, 프로젝트의 구조와 내용을 파악하는 데 필수적인 도구들이다.
- **`read_file` (ReadFile)**: 파일의 내용을 읽어온다. 필요한 부분만 골라 읽는 기능도 지원한다.
- **`list_directory` (ReadFolder)**: 특정 폴더 내의 파일과 디렉토리 목록을 확인한다.
- **`glob` (FindFiles)**: 패턴(예: `**/*.css`)을 사용해 조건에 맞는 파일들을 한꺼번에 찾아낸다.
- **`grep_search` (SearchText)**: 수많은 파일 속에서 특정 키워드나 정규식을 검색한다.

### 🛠️ 실제 작업 및 실행 (Execution)
분석이 끝난 후 실제로 코드에 변화를 주거나 명령을 내리는 도구들이다.
- **`replace` (Edit)**: 파일 내 특정 문구를 정확하게 찾아 다른 문구로 교체한다.
- **`write_file` (WriteFile)**: 새로운 파일을 생성하거나 기존 파일의 전체 내용을 덮어쓴다.
- **`run_shell_command` (Shell)**: 터미널 명령어를 직접 실행한다. (예: 빌드, 테스트, 커밋 등)

### 🤖 전문가 소환 (Sub-agents)
메인 에이전트가 더 깊은 분석이나 특화된 작업이 필요할 때 도움을 요청하는 전문가들이다.
기본 서브 에이전트라고 생각하면 되는데 지금은 하는 역할만 간단하게 알아보고  
더 자세한건 서브에이전트에 대한 설명을 따로 하겠다.
- **`codebase_investigator`**: 코드베이스 전체의 아키텍처와 의존성 관계를 심층 분석한다.
- **`cli_help`**: Gemini CLI 자체의 설정, 명령어, 사용법을 완벽히 꿰고 있는 도우미이다.
- **`generalist`**: 여러 도구를 조합하여 일반적이고 광범위한 작업을 수행하는 해결사이다.

### 🌍 외부 정보 및 전략 (Strategy)
프로젝트 외부의 지식을 가져오거나 복잡한 일의 순서를 정할 때 사용한다.
- **`google_web_search` (GoogleSearch)**: 최신 기술 문서나 스택오버플로우의 해결책을 구글에서 검색한다.
- **`web_fetch` (WebFetch)**: 특정 웹페이지 URL의 텍스트 내용을 그대로 긁어온다.
- **`enter_plan_mode`**: 바로 코드를 고치지 않고, 복잡한 문제 해결을 위한 '전략 기획 단계'로 진입한다.

### 🧠 에이전트 강화 및 소통 (Core)
에이전트의 능력을 확장하거나 사용자와 긴밀히 소통하는 도구들이다.
- **`activate_skill`**: 미리 정의된 전문 워크플로우(Skill)를 에이전트에게 즉시 주입한다.  
(Skill에 대한 이야기는 별도 글에서 설명하고자 한다.)
- **`ask_user`**: 작업 도중 사용자의 의사를 묻거나 추가 정보가 필요할 때 질문을 던진다.
- **`save_memory`**: 중요한 사실이나 사용자 취향을 기억해 두어 다음 대화에서도 활용한다.

## 3. MCP: 외부 Tools 추가

Gemini CLI는 **MCP(Model Context Protocol)**라는 표준을 따르기 때문에
기본제공 Tools외에도 다른 MCP형태의 Tools를 추가할 수 있다.

- **데이터베이스 연동:** SQL 쿼리를 직접 날리는 도구 추가
- **웹 브라우징:** 실제 브라우저를 띄워 화면을 캡처하고 분석하는 도구 추가
- **슬랙/지라 연동:** 메신저로 메시지를 보내거나 티켓을 관리하는 도구 추가

MCP 서버를 설정 파일(`config.yml` 등)에 등록하기만 하면, `/tools` 목록에 새로운 도구가 즉시 나타난다.

유용한 MCP도 있지만, 자칫 여러 MCP를 추가하면 Context가 무거워지기도 하고  
충분한 검토가 되지 않은 MCP라면 보안에 취약할 수 있으니 조심해야 한다. 

---

## 마치며

Tools는 AI에게 '지능'을 넘어 '실행력'을 부여하는 핵심 요소이다.   
어떤 도구들이 있는지 잘 파악하고, 필요하다면 MCP를 통해 나만의 도구를 추가해 보자.   
당신의 AI 에이전트가 매우 뛰어난 생산성을 갖게 될것이다.
